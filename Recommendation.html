Home Feed Recommendation Pipeline

Twomi · MVP V1 · June 2026
Home Screen Recommendation AIA Link MVP

Pipeline Overview

When a user opens the Home feed, the system runs through 6 sequential steps to determine which videos to surface and in what order.

1

User Interest Profile

Determine what the user likes — genres, region, creators

2

Filtering

Remove blocked/reported videos + recently-seen home videos (cooldown)

3

Candidate Pool

Collect the published home videos that survive filtering

4

Scoring

Calculate a score for each video: Relevance + Popularity + Freshness

5

Boosts

Apply AIA Link boost and Featured boost

6

Diversity & Freshness Injection

Prevent repetition, inject exploration content, cap per-creator

Step 1

User Interest Profile

Before searching for videos, the system builds a picture of what the user likes. This profile is the foundation every other step relies on.

Profile Signals
  • Genres — selected by the user in profile (e.g. Music, Dance)
  • Region — taken automatically from the user's account/location, not chosen (e.g. VN, US)
Implicit Signals — Inferred from behavior
  • Favorited avatars → preferred avatars & content Strong
  • Called avatars → avatars you engage with Very strong
  • Watched livestreams → avatars you tune into Strong
  • Completed videos (> 50%) → content you finish Strong
  • Similarity → mood, detected entities & settings of videos you called/watched Content

Cold Start — New Users

Users with no behavior history: use genres from their profile as the only seed. If no genres selected, show top trending content (highest platform-wide Popularity Score).

After the first favorite or call, the interest profile updates immediately.

Interest Profile — established by recent activity (not by time window)

The profile is built from fixed-size lists of the user's most recent activity. No time decay — recency is handled by list size: a new event pushes out the oldest (FIFO).

SourceKeep (MVP, tunable)
Latest favorited avatars5
Latest avatars called5
Latest avatars whose livestream was watched5
Latest completed videos (watched > 50%)10

These lists are the Similarity engaged set and the seed for genre/region inference. The profile is rebuilt — and the feed re-scored — every 20 videos (one feed batch), not on every single event. Until the lists have any entries, the user stays in cold-start (declared genres + trending).

Engaged-set profile — top moods / entities / setting

Pool all videos from the lists above (favorited + called + watched + completed), dedupe → engaged set E. Rank each attribute by frequency over E (all videos weighted equally — the fixed list size already keeps it recent, no time decay) and keep only the top ones:

AttributeHow the profile is built — by appearance frequencyKeep
MoodRank moods by appearance frequency over E (w = videos with that mood / |E|), keep the most frequent → e.g. calm 0.5, happy 0.5, funny 0.3top 3
Detected entitiesRank entities by appearance frequency over E, keep the most frequent → e.g. people 0.8, animals 0.4top 2
SettingTake the most frequent (dominant) value with its share → e.g. indoor 0.6top 1

The kept frequencies are the weights w the Similarity step (4.1) uses (unchanged) — still capped at 0.9. Keeping only top-3 / top-2 / 1 makes the profile small and focused on what the user actually favors; with a larger mood/entity vocabulary this trims to the real preferences. List sizes are tunable.

Example — Returning User
User B, last 2 weeks: favorited avatars [3 Music, 2 Lifestyle], called avatars [3 Music, 1 Comedy], watched 2 Music livestreams, completed 8 videos (mostly Music).

→ Interest Profile: genres: [Music (dominant), Lifestyle, Comedy] · region: VN · engaged_set: recent avatars/videos → feeds Similarity
Step 2

Filtering

Before pooling candidates, apply per-user exclusions. Two kinds of removal: permanent hard filters (table below) and a Seen Cooldown — a video the user has just seen is hidden for a fixed window, then becomes eligible again.

A. Permanent hard filters

Condition Action Reason
User has blocked the creator Remove all of that creator's videos User explicitly blocked
User reported creator / marked "Not interested" Remove all of that creator's videos User does not want this content
User reported a specific video Remove that video only User flagged the content
Creator has blocked the user Remove all of that creator's videos Creator restricted access

B. Seen Cooldown — hide recently-seen, then re-eligible (home videos only)

Once a home video has been seen, it is hidden for that user for a fixed cooldown window — to avoid back-to-back repetition. After the window passes, the video returns to the candidate pool at its normal Score. No permanent removal.

COOLDOWN = 48h (default, tunable) video is excluded while: (now − last_seen_ts) ≤ COOLDOWN after that it is eligible again at full Score
EventDefinitionAction
Display Home video card was shown in the user's feed Update last_seen_ts → start/refresh cooldown
Watch User watched a home video Update last_seen_ts → start/refresh cooldown
Call User called a home video's avatar Update last_seen_ts → start/refresh cooldown
Watch (livestream) User watched a host's livestream Update last_seen_ts on the host's home video → start/refresh its cooldown

Re-seeing a video during its cooldown refreshes last_seen_ts, so the window restarts from the most recent view.

Exception — the livestream itself has no cooldown

A livestream is never put on cooldown — live content is re-joinable and ends on its own, so the stream stays eligible while live. But watching a livestream does start the cooldown on the host avatar's home video (you just saw that avatar's content), so it won't immediately repeat in the feed. The cooldown target is always a home video, never the live stream.

Step 3

Candidate Pool

After filtering, the candidate pool is the published home videos that survive the hard filters and are not in cooldown. Genre pre-filtering is intentionally skipped here — that happens in Scoring. Keeping the pool wide ensures the Diversity step in Step 6 has enough content to pull from.

Step 4

Scoring

Each video receives a score. Videos are sorted highest to lowest. The score has three components. The candidate pool is re-scored every 20 videos (one feed batch), using the freshly rebuilt Interest Profile.

Score = Relevance Score + Popularity Score + Freshness Boost

4.1 — Relevance Score  Range: 0 – 1

How well this video matches the user — three signals: Genre (declared), Region (from account, auto), Similarity (behavioral).

Relevance = 0.45 × Genre Match + 0.15 × Region Match + 0.40 × Similarity
Genre Match
45%
Region Match
15%
Similarity
40%

Each signal uses the same format below — Measures · Scoring · Example. Soft-OR 1 − ∏(1 − x) is used to fold multiple tags into one score: Genre folds its matched genre tags this way; inside Similarity, mood and entities fold their tags this way too. (Similarity then averages its three attributes — see below.)

Genre Match · weight 0.45 · declared

Measures — the video's genre tags (≤3) against the user's genres (1 dominant + secondaries).

Scoring — each matched tag has weight w = dominant 0.9 / secondary 0.7 / no-match 0, combined by soft-OR: Genre = 1 − ∏(1 − w). Breadth rewarded with diminishing returns; dominant is < 1.0 on purpose so multiple matches can compete (all tunable).

Example — user: Music (dom), {Dance, Comedy} (sec). Video [Dance, Comedy, Fitness] → 1 − 0.3² = 0.91; video matching Music only → 0.9 → the broader match wins.

Region Match · weight 0.15 · from account (auto)

Measures — the video's region against the user's region (auto-detected from account, not chosen).

Scoring — match → 1.0 · different → 0.0. Soft signal: a mismatch lowers the score but never removes the video, so cross-region content can still surface.

Example — user VN. Video VN → 1.0 · video US → 0.0 (still shown).

Similarity · weight 0.40 · behavioral

Measures — how much the video resembles your engaged set (recent favorited / called / watched / completed videos) on three attributes: mood & detected entities (each can have several tags) and setting (single value: indoor / outdoor). Captures revealed preference.

Scoring — two steps (engaged-set frequencies come from Step 1).
① Per attribute. Mood & entities have multiple tags → fold them with soft-OR over the video's tags, where each tag's weight w = its frequency in your engaged set, capped at w_max = 0.9 so no single tag alone saturates the attribute to 1.0 (same idea as Genre's dominant 0.9): moodSim = 1 − ∏(1−w), same for entitySim. Setting is a single value → settingSim = the proportion of your engaged set with that setting (no soft-OR needed).
② Combine the three by arithmetic mean: Similarity = (moodSim + entitySim + settingSim) / 3. Cold start (empty engaged set) → 0.

Example — engaged set: moods {calm 0.6, warm 0.3}, entities {guitar 0.5, sofa 0.4}, setting {indoor 0.7, outdoor 0.3}. Video V: moods {calm, energetic}, entities {guitar, mic}, setting indoor.
moodSim = 1 − (1−0.6)(1−0) = 0.6 · entitySim = 1 − (1−0.5)(1−0) = 0.5 · settingSim = 0.7 (share of E that is indoor)
Similarity = (0.6 + 0.5 + 0.7) / 3 = 0.60

MVP uses this attribute proxy; V3 upgrades to cosine over content embeddings (same weight).

4.2 — Popularity Score  Range: 0 – 1

How much the broader platform engages with this video — independent of the viewer's personal taste. Three signals: Favorites (retention), Call-through Rate (Home conversion), Completion (content quality).

Popularity = 0.40 × norm(Favorites) + 0.40 × Call-through Rate + 0.20 × Completion Rate
Favorites
40%
Call-through Rate
40%
Completion Rate
20%

Each signal uses the same format below — Measures · Scoring · Example. Only Favorites is normalised to the candidate pool (it's a raw count); the two rates are already in 0–1 and used as-is.

Favorites · weight 0.40 · retention

Measures — how many distinct users saved this avatar (intent to keep / come back).

Scoring — Favorites = COUNT(DISTINCT user_id) (unique users → resists save/unsave spam), then normalise to the pool: norm = Favorites / max(Favorites in pool). Range 0–1.

Example — video has 200 unique favorites, pool max = 500 → 200 / 500 = 0.40.

Call-through Rate · weight 0.40 · Home conversion

Measures — how well this Home video converts views into calls.

Scoring — CTR = calls_from_home / Home views (raw, ≤ 1). Numerator counts only calls with entry_point='home' & source_video_id = X; denominator = Home impressions of X. Both scoped to Home + this video so the ratio can't exceed 1. Needs entry_point & source_video_id logged on call events.

Example — 1,000 Home views, 200 calls from Home → 200 / 1,000 = 0.20.

Trade-offs: no call depth (5-sec = 5-min call), no anti-spam on repeat callers (swap to distinct callers_from_home if needed), calls from Favorites/Search excluded by design.

Completion Rate · weight 0.20 · content quality

Measures — how fully people watch the (≤10s) Home video — a pure content-quality signal at the video level.

Scoring — Completion = Σ min(watch_i, L) / (views × L), with L = min(video_length, 10s). Watching to the 10s cap = 1.0; bouncing after 2s ≈ 0.2. Raw 0–1.

Example — 1,000 views, total capped watch-time = 6,800s, L = 10 → 6,800 / (1,000 × 10) = 0.68.

Worked example — full Popularity (Favorites_max in pool = 500)
Video B: 200 favorites → 0.40 · CTR 0.20 · completion 0.68
Popularity(B) = 0.40×0.40 + 0.40×0.20 + 0.20×0.68 = 0.160 + 0.080 + 0.136 = 0.376

Video A: 450 favorites → 0.90 · CTR 0.30 · completion 0.80
Popularity(A) = 0.40×0.90 + 0.40×0.30 + 0.20×0.80 = 0.360 + 0.120 + 0.160 = 0.640

4.3 — Freshness Boost

Give new videos a chance before they have engagement data.

Freshness Boost = +0.3 if (now − first_publish_ts) ≤ FRESH_WINDOW = 0 if older than FRESH_WINDOW FRESH_WINDOW = 24h (default) = 48h if creator has AIA Link (see Step 5)

Simple on/off in MVP — no decay curve. Uses the immutable first_publish_ts so deleting and re-posting a video does not reset freshness (anti-abuse).

Worked example — Freshness
At T = 10h after first publish → 10h ≤ 24h → Boost = +0.3 (any creator).
At T = 30h, no AIA → 30h > 24h → Boost = 0.
At T = 30h, creator has AIA Link → window = 48h, 30h ≤ 48h → Boost = +0.3.
Deleted at 20h & re-posted → first_publish_ts unchanged → freshness still counts from the original publish → no reset.

Step 5

Boosts

Applied right after Scoring, on top of the organic Score (additive model). The Diversity & Freshness Injection step (Step 6) then runs on these boosted scores.

AIA Link Boost

Eligibility is binary: AIA_eligible = (AIA account linked). The old AIA Partner Badge (variants A/B/C) was removed 2026-05-24 — the boost now keys off the AIA link only.

Two separate benefits:

  • Extended Freshness Window
    No AIA: Freshness +0.3 applies for 24h.
    With AIA Link: Freshness +0.3 applies for 48h.
  • Permanent Score Boost
    +0.15 added to every video from this creator. No time limit.
Impact at T = 30 hours after posting
No AIA → Freshness = 0 (expired at 24h)
With AIA Link → Freshness = +0.3 (48h window) + permanent +0.15 = +0.45 advantage

Featured Boost

  • Assigned manually by Ops team via CMS (CA-969)
  • Boost value: +0.2 to +1.0 (added to Score)
  • Default when not featured: 0
  • Maximum 3 videos featured simultaneously (configurable)
  • Videos can be tagged with a campaign or event label for tracking
  • Additive: Score_final = Score_organic + AIA_permanent + Featured — does not swallow other boosts

End-to-end Score — additive (Video A, Relevance 0.855 & Popularity 0.640 from 4.1/4.2)

With AIA Link, posted 10h ago: Score = 0.855 (Relevance) + 0.640 (Popularity) + 0.30 (Freshness, 48h window) + 0.15 (AIA permanent) = 1.945 Same video, no AIA, at T = 30h: Score = 0.855 + 0.640 + 0 (freshness expired at 24h) + 0 = 1.495 → 0.45 lower
Step 6

Diversity & Freshness Injection

Pure score-based ranking (including boosts) creates repetition and filter bubbles. This step ensures variety in the final batch.

Per-Creator Cap

Maximum 3 videos from the same creator per batch of 20 videos. If Creator X has the top 8 scores, only their top 3 are kept. The remaining slots are filled by videos from other creators.

Exploration Injection

At least 10% of videos in each feed batch must come from genres outside the user's preferred genres.

Example
User prefers Music and Dance. In a batch of 20 videos:
→ 18 videos from Music, Dance, or avatars matching the user's taste (exploitation)
→ 2 videos from Comedy, Lifestyle, Travel (exploration)

Livestreams

Livestreams don't have favorites/completion data of their own, so they aren't scored like home videos. Instead a livestream inherits its host avatar's Score and adds a Freshness Boost (a live stream is the freshest content there is). Because the host's Score already carries your Relevance to that avatar's content (genre, region, Similarity), a stream whose host matches your taste naturally rises to the top — no manual "Top 3" rule needed. The live stream itself is not subject to the Seen Cooldown (Step 2) and stays eligible while live; watching it does, however, start the cooldown on the host's home video.

Single livestream Score = Host avatar's Score + Freshness Boost Multi livestream Score = max(Host avatars' Scores) + Freshness Boost Host avatar's Score = the Score that avatar's home video earns for this user (Relevance + Popularity) Freshness Boost = +0.3 while live (AIA Link creators: extended 48h window, see Step 5)
Single livestream — 1 host

Score = that host avatar's Score + Freshness Boost.

Multi livestream — 2+ hosts (discussion format)

Score = the highest-scoring host avatar's Score + Freshness Boost, so a strong host can carry the stream. Ties broken toward more hosts.

Example
You've called Avatar A a lot (its home video scores 0.9 for you) and have no history with Avatar B (scores 0.4).
→ Single livestream hosted by A = 0.9 + 0.3 = 1.2
→ Multi livestream hosted by A & B = max(0.9, 0.4) + 0.3 = 1.2

Scope — Home Recommendation ≠ Standalone / Explore cards

The scoring in this doc ranks the organic Home feed only. It does not apply to the Standalone exclusive-content card (CA-2982) or the Explore card (CA-2983) — those are insertion units with their own purchase-based ranking.

Standalone & Explore cards are an insertion layer on top of the already-ranked feed — they don't share the 4.2 / 4.3 / Step 5 formulas.
