Home Video Content
Analysis Pipeline

When a creator saves a home video, the system automatically runs AI analysis and extracts 4 structured signals — Genre, Mood/Aesthetic, Detected Entities, and Setting — used by the recommendation engine, genre classification, and hashtag suggestion, without re-processing the video per feature.
Jira CA-3007 Epic: Home Feed Recommendation
1

User Story

As a product owner,
I want the system to automatically analyze home video content after it is saved,
So that downstream features — genre classification, hashtag suggestion, and recommendation logic — can use rich analysis output without re-processing the video.
2

Context

Without this pipeline, each feature (genre classification CA-2964, hashtag suggestion CA-1008, recommendation engine) would re-process the same video independently — wasteful and inconsistent.

This ticket creates a single shared pipeline that runs once per video and stores a rich output for all consumers. The richer the output, the more dimensions the recommendation engine can use to match the right video to the right user.

This pipeline is backend-only with no user-facing UI. It runs asynchronously and does not block the video save flow.

3

Points of Analysis

When does analysis get triggered? Two cases: videos saved going forward, and the library of videos that already exists.
🆕 New videos

Analysis is triggered automatically after the creator saves the video. The job runs asynchronously and does not block the save flow — the creator and fans see the video as saved immediately.

This is the primary, fully in-scope trigger for the MVP.

🗄️ Existing videos

Videos saved before this pipeline existed have no analysis record. These are processed by batch only.

Batch backfill
A scheduled background job processes the existing library in rate-limited batches during off-peak hours. Prioritize by recency and engagement so the most-watched videos get analyzed first. No lazy or on-demand analysis.

The 4 Analysis Signals

Each signal is extracted from the sampled video frames by the AI vision model and stored for downstream use. Other candidate signals (embedding vector, hook strength, language) are discarded from this scope.
🎭
Signal 1
Genre
1–3 coarse category labels · Detailed classification owned by CA-2964
Genre is the most human-readable signal — a broad category label that describes what the video is about. It is useful for browse pages, filter chips, creator analytics, and as a fallback signal when entity/mood-based recommendations are sparse (e.g., for brand new users with no behavior history). Genre is a coarse label; fine-grained matching relies on entities and mood.
12 supported genres:
MusicDanceComedyEntertainmentBeauty & FashionFoodSportsLifestyleAnimalsGamingEducationTravel

0.71assigned

0.12dropped
Genres assigned: Music, Dance (all scoring above 0.3, top 3 kept). If no genre scores above 0.3 → store null — no tag. There is no default genre.
Error Handling
  • If AI returns no genre scores above 0.3: store null — no tag and log a warning. Downstream features must handle a missing genre gracefully.
🎨
Signal 2
Mood / Aesthetic Tags
Controlled vocabulary · Personalization axis orthogonal to genre and topic
Mood and aesthetic tags describe the emotional tone and visual style of the video — completely separate from what the video is about. Two Music videos can have completely different moods. This axis lets the recommendation engine match users to the feeling of content, not just the subject. Example: a user who watches calm content at night should get more calm content in the evening — regardless of genre.
Controlled vocabulary — the 10 most common mood/aesthetic tags. The AI may only assign tags from this fixed list:
⚡energetic
Fast, vibrant, high-energy
🌊calm
Slow, peaceful, relaxed
😊happy
Bright, upbeat, positive
😂funny
Playful, comedic, lighthearted
🌸wholesome
Warm, cozy, friendly
✨aesthetic
Dreamy, pretty, stylized
🎬cinematic
High contrast, dramatic composition
📻nostalgic
Warm, grainy, lo-fi, retro
💗romantic
Soft, intimate, tender
🌑dark
Moody, low-light, intense
Logic
  • Assign 1 to 3 tags per video, each stored with its own confidence score.
  • Only assign tags whose score exceeds 0.3; keep the top 3 by score. Maximum 3.
  • Tags must come from the fixed controlled vocabulary above — no free-form labels.
  • Derived from: color palette, pacing, lighting style, visual composition, and audio tone signals.
  • Tags are not mutually exclusive — a video can be both cinematic and dark.
Examples
Fast-paced dance video, neon colors, bright lighting → energetic 0.91
Avatar journaling softly in warm café lighting → calm 0.84 + wholesome 0.62
Avatar in a dark high-contrast studio, slow dramatic music → cinematic 0.77 + dark 0.71
Error Handling
  • If no mood/aesthetic tag scores above 0.3: store null — no tag. Downstream features must handle gracefully.
🔍
Signal 3
Detected Entities
Granular list of things visible in frames · Enables fine-grained interest matching
Entities are the specific things the AI can see in the video frames, grouped into three categories: people, animals, and objects. This goes far beyond genre — a single Music video can contain a person, a guitar, a cat, and a studio background simultaneously. Entities cross-cut genre and let the recommendation engine match users on very specific interests.
👤
People
Presence and count of people / avatars in frame
🐾
Animals
Animals with species where detectable (cat, dog, bird…)
📦
Objects
Food, instruments, vehicles, products, furniture, devices…

person · 0.98 cat (animal) · 0.94 pasta (object) · 0.87 pot (object) · 0.79
Logic
  • Detect all entities with a confidence score above the minimum threshold (configurable, default: 0.5).
  • Each entity is tagged with its category: person, animal (with species), or object.
  • No maximum cap on number of entities per video.
  • Store each entity with its label, category, and confidence score.
  • Use case: a user who repeatedly calls avatars in videos containing cats will have "cat" boosted in their interest profile → more cat-containing videos appear in their feed.
Example — Why entities matter beyond genre
Two videos are both genre "Lifestyle". Video A contains: person, cat, book, cozy room. Video B contains: person, gym equipment, sports gear. A user who always engages with videos containing cats gets Video A recommended — not because of genre, but because of the entity match.
Error Handling
  • If no entities are detected above the threshold: store null — no tag. Downstream features must handle this gracefully.
📍
Signal 4
Setting
Indoor / outdoor · Useful for filter UX and context-based personalization
Setting describes the physical environment shown in the video. The core output is a simple indoor / outdoor flag, with an optional scene type for finer detail. This powers filter UX and context-based personalization — a user who consistently watches gym and outdoor content likely prefers active, high-energy settings.
🏠
Indoor
Inside a building or enclosed / virtual space
🌿
Outdoor
Open-air or natural environment
🏠 Indoor scene types (optional)
kitchengymbedroomliving roomstudioofficecar
🌿 Outdoor scene types (optional)
beachstreetparknatureurbanfield
Logic
  • The required output is the indoor / outdoor flag. Scene type is secondary and may be "other".
  • If the background is a virtual or AI-generated environment (common for avatar videos): classify as indoor · studio.
  • If the setting changes across the video: store the dominant setting based on the majority of frames.
Examples
Avatar cooking on a visible stovetop → indoor · kitchen
Avatar walking on a beach at sunset → outdoor · beach
Avatar in front of a solid-color AI-generated background → indoor · studio
Error Handling
  • If indoor/outdoor cannot be determined: store null — no tag.

Marking & Assigning Tags

How the pipeline turns raw AI confidence scores into stored tags. This is fully automated — no human marking step. The rules below summarize the per-signal logic already defined above, in one place.
Signal Output shape Selection rule Threshold / limit If nothing qualifies
🎭Genre List of 1–3 labels, each with a confidence score Highest-scoring genres from the 12-genre vocabulary Score must exceed 0.3 · max 3 tags Store null — no tag
🎨Mood / Aesthetic List of 1–3 tags, each with a confidence score Strongest mood tags by score; not mutually exclusive Score must exceed 0.3 · max 3 tags · fixed vocabulary Store null — no tag
🔍Detected Entities List of {label, category, confidence} Every detected entity, tagged person / animal / object Confidence ≥ 0.5 (configurable) · no max count Store null — no tag
📍Setting Indoor/outdoor flag + optional scene type Dominant environment across frames; virtual bg → indoor · studio Indoor/outdoor required · scene type optional (other allowed) Store null — no tag
All confidence thresholds and limits are configurable without a code deployment. A tag is only ever assigned from its fixed vocabulary — the pipeline never invents new genre, mood, or scene-type labels.

8

Non-Functional Requirements

⏱️
Response Time
Pipeline completes within 5 minutes of video save under normal load.
🔁
Idempotency
Re-triggering for the same video overwrites the existing record without duplicates.
⚙️
Configurable Thresholds
Entity confidence threshold and genre score thresholds must be configurable without code deployment.
📚
Controlled Vocabularies
Genre, mood, and scene-type lists are fixed, versioned vocabularies that downstream features can rely on.
